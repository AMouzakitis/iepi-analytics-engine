### IEPI Analytics Engine

Reference implementation of the Information Entropy Performance Indicator (IEPI), a deterministic BPMN analytics artifact for routing-uncertainty diagnostics and viability assessment in BPMN~2.0 process models with externally specified routing probabilities. This implementation corresponds to the formulation presented in the associated manuscript and is intended for computational reproducibility of the reported results.

Given a predefined process structure, routing-probability assignments, and viability thresholds ((H_{\min}, H_{\max}, \rho_{\min})), the engine computes construct-level diagnostics ((H_N(c), R(c), \kappa(c), f[c])), propagates block-level uncertainty and responsiveness quantities ((U(B), R(B))), and produces a bounded process-level IEPI reporting index. All computations follow directly from the analytical definitions and composition rules presented in the manuscript. The implementation is fully deterministic.

The repository contains the core computation modules (metrics, validation, composition rules, and IEPI scoring), the execution engine, and a reproducibility script (`reproduce_results.py`) that reproduces the reported evaluation results. The current version reproduces the four BPMN evaluation scenarios used in the study:

* Scenario A: XOR authorization workflow
* Scenario B: XOR authorization workflow with additional routing depth
* Scenario C: OR routing workflow with optional compliance review
* Scenario D: LOOP routing workflow with rework and resubmission behavior

The reproducibility script also reproduces the threshold-sensitivity analysis reported in the manuscript.

Routing probabilities must be provided explicitly and are not estimated, inferred, learned, or generated by the engine. The implementation does not perform process mining, BPMN parsing, probability estimation, machine learning, simulation, optimization, or process discovery. Its purpose is deterministic evaluation of routing uncertainty under analyst-specified inputs.

Version v1.0.2 corresponds to the revised implementation aligned with the current manuscript. Earlier archived releases remain available through Zenodo for reproducibility and version tracking.
